TEST PLAN CHALLENGE

HU:Amazon-Alexa

ALCANCE: Para la historia de usuario se realizarán las validaciones y verificaciones donde el usuario pueda buscar el producto "Alexa" en Amazon, y seleccione la tercera opción de la segunda pagina para su posterior validación de disponibilidad y anexo al carrito de compras.
 
Se verificará: 
 
* Se visualicen el campo de búsqueda.
* Se visualice el producto.
* Se visualice la disponibilidad del producto.
Se validará: 
 
* Permita el correcto ingreso a la página.
* Se habilite la opción de anexo al carrito.
* Se pueda seleccionar el producto.
 
 
ESTRATEGIA: Se realizarán escenarios automatizados los cuales estarán enfocados en la ejecución exitosa del ingreso a la pagina de Amazon, el cual se debe buscar el producto "Alexa", seleccionar la tercera opción de la segunda página y así verificar si esta disponible para poder anexar el producto al carrito de compras.
 
CRITERIOS DE DEVOLUCIÓN 
 
* Si el portal web no carga sus componentes.
* Si no se habilitan todos los productos.
 
CRITERIOS DE ACEPTACIÓN 

* Si portal web permita continuar con el ingreso.
* Si se visualiza los componentes de la aplicación.
FUERA DEL ALCANCE 
 
* No se validará el funcionamiento en dispositivos móviles ni en navegadores diferentes a Google Chrome. 
* No se validarán otras funcionalidades que no se encuentren descritas en cada uno de los alcances. 
 
PRERREQUISITOS 
 
* Cada uno de los componentes de la página debe cargar todos sus componentes. 
* Acceso al navegador web Google Chrome. 
* Actualización del navegador web. 

SUPUESTOS Y LIMITACIONES 
 
* Operaciones actualizadas para la ejecución de la prueba.
* El proceso de pruebas no debe contar con restricciones para su realización. 
  
DATOS E INFRAESTRUCTURA 
 
* Disponibilidad en la conexión a la red.

SCOPE: For the user story, validations and verifications will be carried out where the user can search for the "Alexa" product on Amazon, and select the third option on the second page for subsequent validation of availability and attachment to the shopping cart.
 
It will be verified:
 
* The search field is displayed.
* The product is displayed.
* Product availability is displayed.

It will be validated:
 
* Allow the correct entry to the page.
* The option to add to the cart is enabled.
* The product can be selected.
 
STRATEGY: Automated scenarios will be carried out which will be focused on the successful execution of the entry to the Amazon page, which must search for the "Alexa" product, select the third option on the second page and thus verify if it is available to be able to attach the product to the shopping cart.
 
RETURN CRITERIA
 
* If the web portal does not load its components.
* If not all products are enabled.
 
CRITERIA OF ACCEPTANCE

* If the web portal allows you to continue with the income.
